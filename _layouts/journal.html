--- layout: root comment: The page should supply `title` and `date` data: git: [blame, paragraphs, history] --- {%- comment -%} Graceful minimalism for placeholders. A page with no `author` — most often a folder stub created through the GitHub UI, whose whole content is `---\n---` — is not a publishable article: it carries none of the fields the body below assumes (author, title, date), and the contact include alone (contact/.md) would collapse to `contact/.md` and fail the build. Treat author-absence as the stub signal, and honor an explicit `stub: true` for the rare placeholder that has inherited an author from a mount default but still should not render as an article. A stub renders as bare content inside the root chrome; it stays noindex by root's default and, lacking an author, never enters the author indexes.
